# 汎用サンプラー実装計画

作成日: 2026-06-27

この文書は `docs/サンプラー.md` の Phase Plant Sampler 相当機能と自動カット構想を、このリポジトリの現状に合わせて実装へ落とし込むための計画です。Rust 実装、Cargo 設定、既存プラグイン、サブモジュールの中身はこの文書では変更しません。

## 目的

汎用サンプラーを、単一サンプルまたはサンプルセットを MIDI ノートでピッチ追従再生する instrument として実装します。初期段階では「通常サンプラー」を対象にし、Granular Generator は別系統の後続機能として扱います。

最終的に目指す機能は次の通りです。

- WAV/AIFF/FLAC などのサンプル読み込み
- root note、tune、offset、loop start/end、loop crossfade
- Infinite、Sustain、PingPong、Reverse loop
- polyphony、voice stealing、mono retrig/legato
- velocity layer、key range、release trigger
- sampler 内蔵 unison
- 波形表示、marker 編集、preview
- normalize、DC removal、reverse、mono/stereo 変換
- 無音検出、トランジェント検出、BPM 同期による auto trim / auto slice
- native VST3/CLAP と WebCLAP の両対応

## 現状の足場

既に VCSL ピアノ向けのサンプラー実装が入っているため、汎用サンプラーは一から作らず、既存の低レベル部品を汎用化して積み上げます。

確認済みの主な実装面は次の通りです。

- `thirdparty/z-audio-dsp/crates/z-audio-dsp/src/sampler.rs`
  - `SampleBuffer`、`SampleRegion`、`SamplerEngine` がある。
  - mono/stereo PCM、key/velocity range、root key、tune、offset、attack/release trigger、envelope、linear interpolation、voice stealing を持つ。
  - `process` 中の file I/O、decode、log を避ける設計になっている。
- `thirdparty/z-audio-dsp/crates/z-audio-synth/src/vcsl_piano/`
  - VCSL 専用の `VcslPiano`、`VcslSampleBank`、bank binary format、parameter wrapper がある。
  - note event と param event を受けて `SamplerEngine` を駆動している。
- `crates/xtask/src/vcsl_piano.rs`
  - VCSL Keys の SFZ/FLAC から custom bank を作る offline converter がある。
  - SFZ subset parser、FLAC decode、dev bank 生成がある。
- `crates/z-audio-vcsl-piano-plugin/`
  - native VST3/CLAP wrapper がある。
  - disk 上の generated bank を読み、見つからない場合は安全に無音にする。
- `crates/z-audio-webclap-vcsl-piano/`
  - embedded dev bank を `include_bytes!` で持つ WebCLAP instrument がある。
  - UI path 付きの `.wclap` bundle に対応している。

このため初期実装は、VCSL 専用の bank / synth / wrapper をそのまま拡張するのではなく、汎用サンプラー用の型と plugin を追加し、VCSL 側は既存挙動を保つ方針が安全です。

## アーキテクチャ方針

レイヤーを次のように分けます。

```text
sample files / sfz / sampler preset
  -> offline or background asset loading and analysis
  -> generic SamplerBank / SamplerPreset
  -> z-audio-synth generic sampler instrument
  -> z-audio-dsp low-level SamplerEngine
  -> native VST3/CLAP wrapper
  -> WebCLAP wrapper + UI
```

責務の境界:

- `z-audio-dsp`
  - リアルタイム再生だけを担当する。
  - file path、decode、UI state、preset serialization を持たない。
  - `SampleRegion` に loop、direction、xfade、unison に必要な低レベル情報を追加する。
- `z-audio-synth`
  - 汎用サンプラー instrument を追加する。
  - note handling、region selection、slice selection、parameter state、bank load state を持つ。
  - VCSL 専用ロジックとは別 module にする。
- ルートリポジトリ
  - native / WebCLAP wrapper、UI、bundle、file picker、state save/load、xtask converter を担当する。
  - サンプル解析と auto slice は background task または offline tool で実行する。

候補 module:

```text
thirdparty/z-audio-dsp/crates/z-audio-dsp/src/sampler.rs
thirdparty/z-audio-dsp/crates/z-audio-synth/src/generic_sampler/
crates/z-audio-sampler-plugin/
crates/z-audio-webclap-sampler/
crates/xtask/src/sampler_bank.rs
```

## データモデル

### 低レベル DSP

既存 `SampleRegion` に段階的に追加する候補:

| field | 用途 | 初期対応 |
| --- | --- | --- |
| `loop_mode` | off / infinite / sustain / pingpong / reverse | Phase 2 |
| `loop_start_frames` | loop 開始位置 | Phase 2 |
| `loop_end_frames` | loop 終了位置 | Phase 2 |
| `loop_xfade_frames` | loop 境界の等パワークロスフェード | Phase 2 |
| `playback_direction` | forward / reverse | Phase 2 |
| `slice_id` | auto slice 由来 region の識別 | Phase 4 |
| `group_id` | choke / round robin / future group 制御 | 後続 |
| `pan` | region 単位 pan | 後続 |

`SamplerEngine` は現在 region map を持たず、呼び出し側が region を選んで `trigger` する構造です。これは汎用サンプラーでも維持します。region selection や slice selection は synth 側に置いた方が、SFZ、単一サンプル、slice preset の差異を閉じ込めやすいためです。

### 汎用 bank

VCSL 専用の `VcslSampleBank` と `VcslRegionSource` をそのまま公開汎用型に変えると既存 VCSL への影響が大きいため、まずは新規型を追加します。

候補:

```rust
pub struct SamplerBank {
    pub samples: Vec<Arc<SampleBuffer>>,
    pub regions: Vec<Arc<SampleRegion>>,
    pub metadata: SamplerBankMetadata,
}

pub struct SamplerBankMetadata {
    pub name: String,
    pub source_hash: Option<[u8; 32]>,
    pub created_by: String,
    pub format_version: u32,
}
```

binary format は VCSL bank と互換にせず、汎用版として versioned magic を分けます。候補 magic は `ZSMPLBNK` です。VCSL bank は既に `VCSLBANK` として動いているため、混同しない設計にします。

### プリセットとサンプル編集状態

サンプルファイルそのものと、プラグイン状態を分離します。

```text
SamplerBank:
  音声再生に必要な PCM と region map

SamplerPreset:
  root/offset/loop/marker/auto-slice 設定、UI 表示状態、元ファイル参照

PluginState:
  host automation parameter、loaded preset path、missing asset 状態
```

native では sample path を state に保存できますが、WebCLAP では bundle asset または embedded bank を基本にします。外部ファイル参照は host 互換性を確認してから扱います。

## パラメータ設計

既存 `ParamId` では VCSL piano が 180 番台を使っています。実装前に全 enum を確認し、汎用サンプラー用に連番範囲を予約します。候補は 200 番台です。

初期 MVP の parameter:

| Param | 範囲 | automation | 用途 |
| --- | ---: | --- | --- |
| Sample Gain | -48..12 dB | yes | 出力音量 |
| Root Note | 0..127 | yes, stepped | 単一サンプルの基準ノート |
| Tune | -100..100 cent | yes | 微調整 |
| Offset | 0..1 | yes | 再生開始位置 |
| Loop Enabled | bool | yes, stepped | loop on/off |
| Loop Start | 0..1 | yes | loop 開始 |
| Loop End | 0..1 | yes | loop 終了 |
| Loop XFade | 0..200 ms | yes | loop 境界 crossfade |
| Reverse | bool | yes, stepped | 逆再生 |
| Velocity Curve | 0..1 | yes | velocity response |
| Release Time | 0.01..10 s | yes | note off 後の release |
| Stereo Width | 0..1 | yes | mid/side width |
| Max Polyphony | 1..128 | no or stepped | voice 上限 |

Unison と auto slice は MVP 後に追加します。Unison は parameter 数が増えるため、最初は DSP データ構造を壊さず追加できる形だけ先に確保します。

## 実装マイルストーン

### Phase 0: ベースライン固定

目的: 既存 VCSL ピアノの挙動を壊さず、汎用化の基準点を作る。

作業:

- `thirdparty/z-audio-dsp` の current sampler tests を確認する。
- VCSL native / WebCLAP の build と test 手順を記録する。
- `SampleRegion` 拡張前に existing tests を追加または確認する。
- docs 上で VCSL 専用と汎用サンプラーの責務境界を明記する。

完了条件:

- 既存 VCSL piano の unit tests が通る。
- `cargo test --workspace`、wasm check、bundle-webclap の対象を決められる。

### Phase 1: 汎用 SamplerBank と single sample MVP

目的: 1つの WAV/FLAC/AIFF 相当の PCM を bank 化し、MIDI で pitch playback する。

作業:

- `z-audio-synth::generic_sampler` を追加する。
- `SamplerBank` と `SamplerPreset` の最小型を追加する。
- 単一サンプルを 0..127 全キーへ map する region 生成を実装する。
- root note、tune、offset、gain、velocity curve、release time を接続する。
- native plugin crate `z-audio-sampler-plugin` を追加する。
- WebCLAP crate はこの段階では dev bank embedded のみでよい。

完了条件:

- 1つの generated bank で MIDI NoteOn/NoteOff が鳴る。
- pitch ratio が root note と sample rate 差を反映する。
- missing bank 時に plugin がクラッシュせず、明確な status を返せる。
- process 中に file I/O、decode、heap allocation をしない。

### Phase 2: loop / reverse / crossfade

目的: `docs/サンプラー.md` の loop 関連機能を低レベル DSP に入れる。

作業:

- `LoopMode` を追加する。
- forward infinite loop を実装する。
- sustain loop を実装し、NoteOff 後は loop 解除または release stage に遷移する。
- pingpong loop を実装する。
- reverse playback を実装する。
- loop 境界で等パワークロスフェードを入れる。
- loop start/end が逆転、短すぎる、sample 末尾を越える場合の clamp 方針を決める。

完了条件:

- loop 境界で大きな adjacent sample step が出ない unit test がある。
- pingpong / reverse が out-of-range read を起こさない。
- sample rate 44.1 / 48 / 96 kHz で finite output を確認する。

### Phase 3: native UI と state 管理

目的: DAW で実用的にサンプルを読み、再現できる状態にする。

作業:

- native plugin に sample load status を表示する。
- sample path、bank hash、preset parameter を state に保存する。
- bank missing 時は無音安全性と UI warning を両立する。
- 最小波形 view を追加するか、まずは status + numeric controls に絞るか決める。
- file picker と background load の実装方式を決める。

完了条件:

- DAW 再起動後に同じ bank を復元できる。
- bank が移動している場合に明示的に missing 状態になる。
- audio thread で path access や decode が発生しない。

### Phase 4: auto trim / auto slice offline engine

目的: `docs/サンプラー.md` の無音検出とトランジェント検出を、リアルタイム再生と分離して実装する。

初期対応:

- RMS 無音検出
- zero crossing snap
- minimum slice length
- marker merge / dedupe
- slice preview 用の region 生成

次段階:

- fast/slow envelope 差分による transient detection
- spectral flux
- BPM 推定または固定 BPM grid slice
- slice ごとの normalize / fade in / fade out

完了条件:

- 解析処理は background / offline のみで実行される。
- audio thread は確定済み marker / region を読むだけ。
- synthetic fixture で expected slice positions を検証する。
- 実サンプル `docs/samples/piano.wav` などで手動確認できる。

### Phase 5: WebCLAP sampler MVP

目的: WebCLAP でも汎用サンプラーの最小音出しと UI 表示を成立させる。

作業:

- `z-audio-webclap-sampler` crate を追加する。
- embedded small bank で初期ロードする。
- `plugin.json` に instrument / sampler / stereo feature を設定する。
- UI は host parameter、load status、簡易 keyboard visual に絞る。
- `.wclap/assets/` から bank を読む方式は調査後に追加する。

完了条件:

- `cargo check --target wasm32-unknown-unknown -p z-audio-webclap-sampler` が通る。
- `cargo xtask bundle-webclap --release` で tarball が生成される。
- tarball に `module.wasm`、`plugin.json`、必要な `ui/` が含まれる。
- WebCLAP host で load、MIDI input、parameter set/get を確認できる。

### Phase 6: unison / mono mode / editor features

目的: Phase Plant ライクな演奏・音作り機能を追加する。

作業:

- sampler voice 内 subvoice 構造を追加する。
- detune、spread、blend を実装する。
- mono retrig / legato を追加する。
- sample editor の normalize、DC removal、reverse、mono/stereo 変換を offline processor として実装する。
- 操作後の結果を destructive edit ではなく preset/bank 派生として保存する。

完了条件:

- Unison voice count を上げても output が finite で過大ゲインにならない。
- mono legato が NoteOn sequence で不要な attack retrigger を起こさない。
- editor 処理は元ファイルを破壊せず、生成 bank または preset に記録される。

### Phase 7: 高音質化と大容量対応

目的: 実用音質と大容量サンプル対応を改善する。

候補:

- 4-point Hermite / cubic interpolation
- windowed-sinc interpolation
- pitch-up 時の anti-aliasing
- sample deduplication
- lazy decode / streaming
- memory mapped file
- sample cache eviction

完了条件:

- CPU 使用率と音質差の比較がある。
- WebCLAP で wasm サイズ、load time、memory 使用量を記録する。
- native で大容量 bank の初回ロード時間を記録する。

## テスト計画

### DSP / submodule

- region match: key / velocity / trigger が正しい。
- pitch ratio: root note、tune、source sample rate、output sample rate が反映される。
- offset: sample 開始位置がずれる。
- loop: infinite / sustain / pingpong / reverse が範囲外 read を起こさない。
- xfade: loop 境界と sample end で急激な sample step が出ない。
- voice stealing: releasing voice 優先、steal fade による click 回避。
- no allocation during process: 既存方針に合わせて確認する。
- finite output: 44.1 / 48 / 96 / 192 kHz、block size 1 / 64 / 128 / 512。

### asset / preset

- WAV/AIFF/FLAC decode の sample rate、channel、frame count が一致する。
- generated bank が round-trip する。
- source hash が安定する。
- unsupported metadata は warning として保持する。
- missing sample path が明示的に error になる。

### auto slice

- 無音区間中心に marker が置かれる。
- minimum slice length 未満の marker が merge される。
- zero crossing snap が許容範囲内で動く。
- transient fixture で onset marker が ±指定 ms に入る。
- long file 解析で処理時間とメモリ使用量を記録する。

### plugin / bundle

- `cargo test --workspace`
- `cargo test --manifest-path thirdparty/z-audio-dsp/Cargo.toml`
- `cargo xtask bundle z-audio-sampler-plugin --release`
- `cargo check --target wasm32-unknown-unknown -p z-audio-webclap-sampler`
- `cargo xtask bundle-webclap --release`
- tarball 内容確認
- host smoke test: MIDI input、automation、state save/load、missing bank、sample-rate change

## リスクと先に決めること

- 汎用 bank format を VCSL bank と統合するか、別 format にするか。
- native plugin state に absolute path を保存するか、copy/import した bank を plugin data に持つか。
- WebCLAP で外部 sample を扱えるか。embedded bank だけだと汎用性が弱い。
- WAV の `smpl` chunk、AIFF marker、SFZ、JSON preset のどこまでを初期対応するか。
- loop crossfade を再生時に計算するか、bank 生成時に preprocess するか。
- Unison を `SamplerEngine` 内で扱うか、synth 側で複数 trigger として扱うか。
- auto slice の高精度化に FFT / ML を入れるか。MVP では RMS と envelope 差分までに絞るのが安全。
- 大容量 sample の full preload は native では許容できても WebCLAP では厳しい可能性がある。

## 推奨する初期 PR 範囲

最初の実装 PR は次に絞ります。

- 汎用 `SamplerBank` / `SamplerPreset` の最小追加。
- 単一 sample bank を生成する offline helper。
- `z-audio-synth::generic_sampler` の NoteOn/NoteOff / Param 対応。
- native `Z Audio Sampler` の最小 wrapper。
- root note、tune、offset、gain、release、velocity curve のみ。
- loop、auto slice、unison、波形 editor は後続に分離。

この範囲なら、既存 VCSL サンプラーを壊さず、汎用サンプラーのファイル形式、DSP 境界、plugin 境界を早い段階で検証できます。
